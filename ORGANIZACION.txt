GUÍA DE REORGANIZACIÓN DEL PROYECTO (CALBRU GAMES)

Para organizar el proyecto y evitar que App.tsx sea un archivo gigante, seguiremos esta estructura basada en las carpetas que has creado. Aquí tienes los archivos que debes crear y qué contenido de App.tsx debes mover a cada uno.

---

1. CARPETA: src/types/ (Recomendado crear esta carpeta)
   - Archivo: index.ts
   - Contenido: Mueve aquí las interfaces y tipos (Genre, Particle, Step, OptionsMap, Page).

2. CARPETA: src/constants/ (Recomendado crear esta carpeta)
   - Archivo: gameData.ts
   - Contenido: Mueve aquí la constante GENRES y la lista RANDOM_COLORS.

3. CARPETA: src/Landing/
   - Archivo: LandingPage.tsx
   - Contenido: Todo el código dentro de la función `renderHomePage`.
   - Propósito: Es la página de inicio que los usuarios ven al entrar.

4. CARPETA: src/Selection-Cauldron/
   - Archivo: SelectionPage.tsx
   - Contenido: Todo el código dentro de la función `renderSelectionStep`.
   - Propósito: La pantalla donde el usuario elige su "poción base" (el género del juego).

5. CARPETA: src/Adding-Ingredients/
   - Archivo: ConfiguratorPage.tsx
     - Contenido: La función `renderConfiguratorStep`. Es el contenedor principal de esta sección.
   - Archivo: Cauldron.tsx
     - Contenido: El JSX de `.center-dashboard` (el círculo central y la imagen del caldero).
   - Archivo: OptionsShelf.tsx
     - Contenido: La función `generateOptionsGrid` que crea las estanterías de las esquinas.
   - Archivo: ParticleEffect.tsx
     - Contenido: La lógica de `createParticle` y el componente visual `.falling-particle`.

6. CARPETA: src/Intranet/
   - Archivo: IntranetPage.tsx
   - Contenido: (Vacío por ahora)
   - Propósito: Aquí irá el panel de administración o gestión interna que planees crear más adelante.

---

¿CÓMO QUEDARÍA App.tsx?
App.tsx pasaría a ser solo el "Director de Orquesta":
- Importará todos los componentes anteriores.
- Mantendrá los estados principales (page, currentStep, selectedGenre, selections).
- Tendrá las funciones que cambian esos estados y las pasará como "props" a los componentes hijos.

Beneficios:
- Si quieres cambiar algo del caldero, vas directo a Adding-Ingredients/Cauldron.tsx.
- El código es mucho más legible y fácil de mantener.
- Evitas errores al tocar partes del código que no deberías.
